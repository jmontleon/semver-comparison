Modal Migration (Deprecated → Composable)

What changed in PF6

The old Modal is moved to @patternfly/react-core/deprecated. The new composable Modal API uses ModalHeader, ModalBody, ModalFooter as children.

pf-codemods approach

The branch uses a hybrid approach:

  • Modal and ModalVariant imported from @patternfly/react-core/deprecated (19 files)
  • ModalHeader, ModalBody, ModalFooter imported from @patternfly/react-core
  • Modal content restructured to use the composable pattern (children instead of actions prop)

Before (pf5)

<Modal title="Edit" isOpen onClose={...} actions={[
  <Button key="save" ...>Save</Button>,
  <Button key="cancel" ...>Cancel</Button>,
]}>
  {children}
</Modal>

After (pf-codemods)

import { Modal } from '@patternfly/react-core/deprecated';
import { ModalHeader, ModalBody, ModalFooter } from '@patternfly/react-core';

<Modal isOpen onClose={...}>
  <ModalHeader title="Edit" />
  <ModalBody>{children}</ModalBody>
  <ModalFooter>
    <Button key="save" ...>Save</Button>
    <Button key="cancel" ...>Cancel</Button>
  </ModalFooter>
</Modal>

Assessment

This is a valid interim migration strategy per PF6 docs. The deprecated import path is the correct first step. Full migration to the new Modal API (not from deprecated) would be the ideal final state, but the approach taken is functional and forward-compatible.

DragDrop → DragDropSort Migration

What changed in PF6

DragDrop, Draggable, Droppable are deprecated and moved to @patternfly/react-core/deprecated. The replacement is DragDropSort from @patternfly/react-drag-drop.

pf-codemods results

The ManageColumnsModal was substantially refactored to use the new DragDropSort API:

// Before
import { DragDrop, Draggable, Droppable } from '@patternfly/react-core';
<DragDrop onDrop={onDrop}>
  <Droppable hasNoWrapper>
    <DataList>
      {items.map(item => <Draggable>...</Draggable>)}
    </DataList>
  </Droppable>
</DragDrop>

// After
import { DragDropSort } from '@patternfly/react-drag-drop';
<DragDropSort items={items} onDrop={onDrop} variant="DataList">
  <DataList>...</DataList>
</DragDropSort>

The onDrop callback signature changed from (source, dest) => boolean to (event, items, oldIndex, newIndex) => void, which was correctly adapted.
