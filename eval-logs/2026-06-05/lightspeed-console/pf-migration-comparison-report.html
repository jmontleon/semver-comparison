6.1 Package Upgrades

📦 PatternFly Package Versions semver/goose wins

PF6 Migration Requirement: The official upgrade guide states: "Update all PatternFly dependencies to v6 in package.json" as Step 1, before running any codemods.

Companion Dependency Upgrades (semver/goose only)

The semver/goose branch also upgrades the OpenShift Console SDK from 1.0.0 to 4.23.0-prerelease.2. This SDK version has peer dependencies that require React 18, react-redux 9, react-i18next 16, react-router 7, redux 5, and redux-thunk 3:

// SDK 4.23.0-prerelease.2 peerDependencies:
{
  "@patternfly/react-topology": "~6.5.0",
  "react": "^18.3.1",
  "react-i18next": "~16.5.8",
  "react-redux": "~9.2.0",
  "react-router": "~7.13.1",
  "redux": "~5.0.1",
  "redux-thunk": "~3.1.0"
}

This means the broader dependency upgrades in semver/goose are justified — they satisfy the SDK's peer dependency requirements. However, note that PF6 itself accepts React 17, 18, or 19 as peers.

⚠️ Concern: react-modal downgrade

The semver/goose branch downgrades react-modal from ^3.16.3 to ^1.4.0 (resolves to 1.9.7). This is a major version downgrade. The app uses react-modal's isOpen, onRequestClose, className, overlayClassName, and ariaHideApp props — all of which exist in v1.9.7. The build succeeds, but this is likely an AI error and should be fixed to ^3.16.3 or later.

🪟 Modal Migration semver/goose wins

PF6 introduces a composable Modal pattern with ModalHeader, ModalBody, and ModalFooter sub-components. The old Modal API is moved to @patternfly/react-core/deprecated.

pf-codemods approach:

// ConfirmationModal.tsx — imports from deprecated path
import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';
// ❌ BUILD ERROR: PF5 packages don't have /deprecated exports

The codemod correctly identified the need to move Modal to deprecated, but since packages weren't upgraded, this import doesn't resolve.

semver/goose approach:

// ConfirmationModal.tsx — full PF6 composable Modal
import { Button, Modal, ModalBody, ModalFooter, ModalHeader, ModalVariant } from '@patternfly/react-core';

<Modal variant={ModalVariant.small} isOpen={isOpen} onClose={onDecline}>
  <ModalHeader title={title} titleIconVariant="warning" />
  <ModalBody>{message}</ModalBody>
  <ModalFooter>
    <Button onClick={onConfirm}>{confirmText}</Button>
    <Button variant="link" onClick={onDecline}>{declineText}</Button>
  </ModalFooter>
</Modal>

Fully migrated to the PF6 composable pattern. Also applied ModalBody/ModalHeader wrapping in AttachEventsModal and NewChatModal.
