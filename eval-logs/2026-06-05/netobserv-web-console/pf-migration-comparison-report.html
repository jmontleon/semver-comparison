Key Insight

The Codemods branch represents a valid first step — running npx @patternfly/pf-codemods@latest --v6 --fix — but it is only one step of the multi-step migration process described in the official PF6 upgrade guide. The guide explicitly states that codemods handle some transformations and that manual work is required afterward, including: package version bumps, CSS class/token updates, structural API changes, and running the cleanup codemod to remove data-codemods attributes.

The Goose branch completed most of these follow-up steps, making it a far more complete migration. However, Codemods did catch a few things Goose missed (PageSection hasBodyWrapper, MastheadToggle restructuring), suggesting the ideal workflow would be: run codemods first, then apply Goose-style AI-assisted fixes, then manually review remaining gaps.
