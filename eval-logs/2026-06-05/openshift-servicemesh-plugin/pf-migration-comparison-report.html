Analysis

The pf-codemods branch ran the codemod tool but did not update package.json — the single most critical step. This means the code uses PF6 APIs against PF5 packages, which is fundamentally broken. The codemods ran the migration transforms without first installing PF6, violating the official upgrade procedure.

The goose branch correctly updated all 6 PF packages to PF6 versions, added the 17 Victory chart peer dependencies (required by @patternfly/react-charts@8), and removed the deprecated patternfly-charts-theme-dark.css import.

Winner: goose — Package upgrade is the foundation of the entire migration. This alone accounts for a large portion of the overall gap.

Modal — Deprecated vs. Composable API

What changed in PF6

The old props-based Modal (title, actions props) is deprecated. PF6 provides a new composable Modal with <ModalHeader>, <ModalBody>, <ModalFooter> sub-components. The codemod only moves imports to /deprecated; actual migration to the composable API is manual work.

Before (PF5 — old API)

<Modal title="Debug info" variant={ModalVariant.medium}
  isOpen={isOpen} onClose={onClose}
  actions={[<Button key="c">Copy</Button>]}>
  Body content
</Modal>

After (PF6 — composable API)

<Modal variant={ModalVariant.medium}
  isOpen={isOpen} onClose={onClose}>
  <ModalHeader title="Debug info" />
  <ModalBody>Body content</ModalBody>
  <ModalFooter><Button>Copy</Button></ModalFooter>
</Modal>
