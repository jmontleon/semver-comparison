semver wins Modal Migration (Major API Change)

What changed in PF6

The PF5 Modal component (prop-based: <Modal title="..." actions={[...]}>) was deprecated. The new PF6 Modal uses a compositional pattern with ModalHeader, ModalBody, and ModalFooter children. The old Modal is still available under @patternfly/react-core/deprecated but will be removed in a future version.

PF5 pattern (before)

<Modal
  title="Delete Storage Pool"
  variant={ModalVariant.small}
  isOpen={isOpen}
  onClose={closeModal}
  actions={[<Button onClick={onDelete}>Delete</Button>]}
>
  Are you sure you want to delete this pool?
</Modal>

PF6 pattern (new API — semver approach)

<Modal variant={ModalVariant.small} isOpen={isOpen} onClose={closeModal}>
  <ModalHeader title="Delete Storage Pool" />
  <ModalBody>
    Are you sure you want to delete this pool?
  </ModalBody>
  <ModalFooter>
    <Button onClick={onDelete}>Delete</Button>
  </ModalFooter>
</Modal>

Deprecated pattern (pf-codemods approach)

// Only changed the import path:
import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';
// Component usage unchanged — still uses PF5 prop-based API

  • Package.json was not updated — still references PF5 packages
  • yarn.lock was not updated — yarn install would install PF5

  • 11 token references point to t_temp_dev_tbd (renders hot pink)
  • Zero CSS/SCSS files were migrated (511 pf-v5 references remain)
  • Modals use deprecated compatibility shim instead of new API
  • ~70% of migration work still needs to be done

This branch could serve as a useful starting point for migration if combined with dependency upgrades and manual token/CSS work, but it is not a viable migration on its own.
