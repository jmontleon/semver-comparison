🔲 EmptyState Migration Winner: semver/goose

What changed in PF6

Per the official PatternFly Upgrade Guide: EmptyStateHeader and EmptyStateIcon are removed. Their functionality is absorbed into the EmptyState component via the titleText, headingLevel, and icon props. EmptyStateBody remains valid. EmptyStateActions replaces EmptyStateSecondaryActions.

// IncidentsTable.jsx — STILL USES EmptyStateIcon ❌

Partially migrated. Uses <Title> inside titleText (wrong — PF6's titleText accepts a string/ReactNode for the text content, not a <Title> wrapper). Still uses EmptyStateHeader and EmptyStateIcon in 2 files.

🪟 Modal Migration Winner: semver/goose

What changed in PF6

In PF6, there are two paths for Modal: (1) use the promoted new Modal from @patternfly/react-core with composable ModalHeader, ModalBody, ModalFooter sub-components, or (2) use the old Modal from @patternfly/react-core/deprecated. The new promoted Modal is the recommended path.

pf-codemods-060526-2202

// SilencesUtils.tsx — Imports from /deprecated ❌

} from '@patternfly/react-core/deprecated';

// But /deprecated doesn't exist in PF5!
// This causes TS2305 build error

Moved Modal to /deprecated import path, which doesn't exist in PF5 packages. The old title prop pattern is preserved. Does not use the promoted PF6 Modal with composable sub-components.

❌ DO NOT MERGE: pf-codemods-060526-2202

This branch is an incomplete, non-building intermediate state. It represents only the raw output of @patternfly/pf-codemods without the essential prerequisite (package upgrades) or follow-up work (token resolution, CSS migration, remaining component migration). It could serve as a reference for what the codemod tool identifies, but is not mergeable as-is.

Why the score gap is so large (92 vs 24)

The pf-codemods branch committed the cardinal mistake of a dependency migration: it changed the code for the new version without actually upgrading the dependencies. This creates a fundamentally broken state where every codemod-applied change becomes a type error. While the scope is clean and no behavior was changed, none of the changes are functional because the foundation (packages) was never updated.

The semver/goose branch, despite a broader scope and some scope creep from included bugfixes, delivers what matters: a working, buildable, type-safe PatternFly 6 migration with proper package versions, correct component APIs, resolved design tokens, and migrated CSS classes.
