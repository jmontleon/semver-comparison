PageSection variant="light" Removal Winner: goose

What Changed in PF6

In PF6, PageSection only accepts variant="default" or variant="secondary". The "light" variant is removed. The official codemods replace variant="light" with removal of the prop (since default is now the light equivalent).

pf-codemods-060426-2245

// Replaces variant="light" with hasBodyWrapper={false}
<PageSection hasBodyWrapper={false} >

⚠️ hasBodyWrapper={false} is a different prop that controls whether PageSection renders a body wrapper div. While it works, it changes the semantic meaning. The correct migration is to simply remove variant="light" or use variant="default".

semver/goose/060426-2234

// Some use variant="default", some just remove the prop
<PageSection variant="default">  // TracesPage
<PageSection>                    // QueryBrowser, TraceDetailPage

✅ Uses the correct PF6 approach. variant="default" and plain <PageSection> are both semantically correct replacements for variant="light".

Charts API Type Safety (getThemeColors) Winner: goose

What Changed in PF6

In PF6's @patternfly/react-charts v8, the getThemeColors() return type changed. The .chart.colorScale property became optional and its type changed from string[] to ColorScalePropType.

pf-codemods-060426-2245

// ❌ Uses old type assumptions (causes 4 TS errors)
const palette = getThemeColors(
  ChartThemeColor.multiUnordered,
).chart.colorScale.flatMap((cssColor) => {
  ...
});

❌ 4 TypeScript errors: TS2532 (Object possibly undefined × 2), TS2339 (flatMap doesn't exist on ColorScalePropType), TS7006 (implicit any).

semver/goose/060426-2234

// ✅ Adds optional chaining and type assertion
const palette = (getThemeColors(
  ChartThemeColor.multiUnordered,
).chart?.colorScale as string[] ?? []).flatMap(
  (cssColor: string) => {
  ...
});

✅ 0 TypeScript errors. Handles optional chaining, type assertion, and fallback correctly.

Select onSelect Type Signature Winner: goose

What Changed in PF6

The Select component's onSelect callback value parameter type changed. In PF5 it was string | number | undefined; in PF6 it uses SelectOptionProps['value'].

pf-codemods-060426-2245

// ❌ Old type signature
const onSelect = (
  _event: React.MouseEvent<...> | undefined,
  value: string | number | undefined,
) => { ... };

⚠️ Not updated — still uses PF5 type. Works but doesn't match PF6 types.

semver/goose/060426-2234

// ✅ Updated type signature
const onSelect = (
  _event: React.MouseEvent<...> | undefined,
  value: SelectOptionProps['value'] | undefined,
) => { ... };

✅ Updated in both BasicSelect.tsx and TypeaheadSelect.tsx to use PF6-correct type.
