🪟 Modal Migration

PF6 change: Modal now has a composable API with ModalHeader, ModalBody, ModalFooter. The old Modal with title/actions props is deprecated and fully removed from PF6.

pf-codemods approach

Moved Modal import to @patternfly/react-core/deprecated, kept old API.

Problem: PF6's deprecated path doesn't actually export Modal — it was fully removed, not deprecated.

semver/goose approach

Migrated all 5 modal files + TabModal to new composable API with ModalHeader/ModalBody/ModalFooter.

Actions moved from array prop to ModalFooter children.

Before (PF5)

<Modal
  title="Delete"
  actions={[
    <Button key="confirm">Delete</Button>,
    <Button key="cancel">Cancel</Button>
  ]}
>
  Content here
</Modal>

After — semver/goose (PF6 ✅)

<Modal>
  <ModalHeader title="Delete" />
  <ModalBody>Content here</ModalBody>
  <ModalFooter>
    <Button>Delete</Button>
    <Button>Cancel</Button>
  </ModalFooter>
</Modal>

🏆 Recommendation: Use semver/goose/060526-0136

The semver/goose branch is a comprehensive, building migration that upgraded all packages, updated all source code, fixed all CSS references, handled React 18 compatibility, and used the correct PF6 APIs including the new composable Modal, DragDropSort, and proper design tokens.

The pf-codemods branch is fundamentally broken: it ran PF6 codemods to transform source code but never upgraded the package.json dependencies. The result is PF6 syntax running against PF5 packages, producing 30 build errors and leaving all CSS references untouched. It represents only the first (incomplete) step of a multi-step migration process.

Margin of victory: 54 points (89.5 vs 35.5) — semver/goose wins 5 of 6 categories, losing only Functional Preservation by 0.5 points (because it made more structural changes as part of actually completing the migration).
