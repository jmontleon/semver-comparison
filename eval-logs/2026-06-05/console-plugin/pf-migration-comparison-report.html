EmptyState Restructuring

PF6 changed EmptyState composition. In v6.5.1, EmptyStateHeader still exists and accepts titleText, icon, headingLevel props. The EmptyState component itself does NOT accept these props directly.

// PF5 (original)
<EmptyState>
  <EmptyStateHeader titleText="..." icon={<EmptyStateIcon icon={BanIcon} />} headingLevel="h4" />
</EmptyState>

// What BOTH branches did (incorrect for v6.5.1):
<EmptyState headingLevel="h4" titleText="..." icon={BanIcon}></EmptyState>

// Correct for PF6 v6.5.1:
<EmptyState>
  <EmptyStateHeader titleText="..." icon={<EmptyStateIcon icon={BanIcon} />} headingLevel="h4" />
</EmptyState>

Both branches have identical EmptyState issues in error.tsx. However, Goose at least produces a visible compile error (2 errors) while Codemods' errors are masked by 78 other errors.

PageSection variant="light"

PF6 removed variant="light". The official codemod fix is to remove the prop entirely (default is now "default", which gives a light background).

// PF5
<PageSection variant="light">

// Goose (correct approach):
<PageSection variant="default">

// Codemods (INCORRECT — hasBodyWrapper is unrelated):
<PageSection hasBodyWrapper={false}>

Source: Official pf-codemods rule pageSection-update-variant-values removes the prop entirely. hasBodyWrapper controls <PageBody> wrapper rendering, not theming.
