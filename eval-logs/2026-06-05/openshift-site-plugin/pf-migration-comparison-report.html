🔲 EmptyState Component Migration — Winner: semver/goose

What changed in PF6

In PF6, EmptyStateHeader and EmptyStateIcon are no longer exported. The EmptyState component now renders the header/icon internally. You pass titleText, headingLevel, and icon as props directly on <EmptyState>. The icon prop takes the icon component reference (e.g., CubesIcon), not a JSX element.

    headingLevel="h4"
    icon={<EmptyStateIcon icon={CubesIcon} />}
  />
  <EmptyStateBody>...</EmptyStateBody>
</EmptyState>

After (PF6 — correct)

<EmptyState
  variant={EmptyStateVariant.xl}
  titleText="No site configured"
  headingLevel="h4"
  icon={CubesIcon}
>
  <EmptyStateBody>...</EmptyStateBody>
</EmptyState>

📋 Modal Component Migration — Winner: semver/goose

What changed in PF6

The old monolithic Modal is deprecated in PF6. The new composable Modal (promoted from @patternfly/react-core/next) uses <Modal> with child components like <ModalHeader>, <ModalBody>, <ModalFooter>. The deprecated Modal can still be imported from @patternfly/react-core/deprecated as a bridge. The showClose prop is removed (close button is now always present in ModalHeader). The hasNoBodyWrapper prop is removed.

Why Not pf-codemods?

The codemods branch has value as a reference for specific code transformations but is not viable as a merge target.
