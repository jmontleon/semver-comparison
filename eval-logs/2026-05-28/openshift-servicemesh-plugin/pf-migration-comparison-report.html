🔍 Detailed Analysis

🚪 Deprecated Modal Migration (Highest Impact) Winner: Goose

What changed in PF6

The old Modal component (with title, actions, header props) was deprecated. The new PF6 Modal uses a composable pattern with <ModalHeader>, <ModalBody>, and <ModalFooter> sub-components. The old Modal was moved to @patternfly/react-core/deprecated; the new one lives at @patternfly/react-core.

Before (PF5 deprecated pattern)

import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';

<Modal
  title="Debug information"
  variant={ModalVariant.medium}
  isOpen={isOpen}
  onClose={onClose}
  actions={[<Button onClick={onClose}>Close</Button>]}
>
  <p>Content here...</p>
</Modal>

After (PF6 composable pattern)

import { Modal, ModalBody, ModalFooter, ModalHeader, ModalVariant } from '@patternfly/react-core';

<Modal
  variant={ModalVariant.medium}
  isOpen={isOpen}
  onClose={onClose}
>
  <ModalHeader title="Debug information" />
  <ModalBody>
    <p>Content here...</p>
  </ModalBody>
  <ModalFooter>
    <Button onClick={onClose}>Close</Button>
  </ModalFooter>
</Modal>

🔘 Button Icon Prop Migration Winner: Goose

What changed in PF6

For variant="plain" buttons, icons must now be passed via the icon prop instead of as children. The official PF codemods handle this automatically for variant="plain" buttons.

// PF5:
<Button variant="plain"><CopyIcon /></Button>

// PF6:
<Button variant="plain" icon={<CopyIcon />} />

⚠️ Known Bug in semver/goose: ValidationStack.tsx

The StackItem component prop expects a React.ElementType (like 'div', 'span', or a component class) — it's used internally as <Component ...>. The Goose branch incorrectly passes:

  • A JSX element (<Validation .../>) as component — this will fail at runtime because React can't instantiate an already-rendered element
  • A string ("Istio validations") as component — this creates an HTML element named <Istio validations> which is invalid HTML
// BUG — component receives a React element instance, not a component type
<StackItem component={<Validation severity={check.severity} message="..." />} />

// FIX — use children (the original pattern was correct)
<StackItem><Validation severity={check.severity} message="..." /></StackItem>

Impact: Medium — affects tooltip validation displays. Fix is trivial: revert to the original children-based pattern.
