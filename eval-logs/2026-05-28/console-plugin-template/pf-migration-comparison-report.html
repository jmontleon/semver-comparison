🧩 Component API Changes — PageSection → Page (INCORRECT)

What Changed in PF6

PageSection was NOT removed or renamed in PatternFly 6. It remains a valid component at @patternfly/react-core. The Page component is a completely different component — it's the top-level page wrapper, not a content section.

Evidence: PageSection exists in PF6

export * from './PageSection';   // ✅ PageSection is still exported in PF6

// The Page directory contains both:
// - Page.js (top-level page wrapper)
// - PageSection.js (content section within a page)

Before (main — CORRECT)

import { Content, PageSection } from '@patternfly/react-core';
// ...
<PageSection>
  <Content component="p">...</Content>
</PageSection>

After (pf-codemods — INCORRECT)

import { Content, Page } from '@patternfly/react-core';
// ...
<Page>
  <Content component="p">...</Content>
</Page>
Semantic breakage: PageSection provides sectioned content styling with padding and background. Page is the full page shell/wrapper. In an OpenShift Console plugin, the console already provides the outer <Page> component. Wrapping content in a nested <Page> will:
  • Create a nested page layout (page-within-a-page)
  • Lose the section padding and styling that PageSection provides
  • Potentially break the overall page layout structure
