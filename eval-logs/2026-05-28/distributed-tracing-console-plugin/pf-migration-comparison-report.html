📄 PageSection hasBodyWrapper pf-codemods WINS

What changed in PF6

PF6 introduced hasBodyWrapper (default: true) on PageSection. When true, children are automatically wrapped in a <PageBody> element. PF5 did not have this behavior — children were rendered directly.

The official PF6 migration guide and the pf-codemods tool both flag this: for existing code that doesn't expect the PageBody wrapper, hasBodyWrapper={false} must be set to preserve PF5 rendering behavior.

Before (main branch)

// TracesPage.tsx — 6 instances like this:
<PageSection>
  <Title headingLevel="h1">{t('Traces')}</Title>
</PageSection>

// QueryBrowser.tsx — 2 instances like this:
<PageSection className="mui-pf-theme">
  <Split hasGutter>...</Split>
</PageSection>

// TraceDetailPage.tsx — 1 instance:
<PageSection>
  <Breadcrumb>...</Breadcrumb>
</PageSection>

After (pf-codemods branch)

// TracesPage.tsx — all 6 fixed:
<PageSection hasBodyWrapper={false}>
  <Title headingLevel="h1">{t('Traces')}</Title>
</PageSection>

// QueryBrowser.tsx — both fixed:
<PageSection hasBodyWrapper={false} className="mui-pf-theme">
  <Split hasGutter>...</Split>
</PageSection>

// TraceDetailPage.tsx — fixed:
<PageSection hasBodyWrapper={false}>
  <Breadcrumb>...</Breadcrumb>
</PageSection>
