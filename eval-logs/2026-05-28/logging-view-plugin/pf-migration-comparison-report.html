🪟 Modal Migration Winner: semver/goose

What changed in PF6

PF6 introduced a new Modal component (composable with ModalHeader, ModalBody, ModalFooter). The old Modal (with title prop, ModalVariant enum) was moved to @patternfly/react-core/deprecated.

The new Modal does not accept a title prop — titles are provided via the <ModalHeader> child component.

Base branch state (bug)

The main branch has a broken hybrid: imports from the new API path but uses the old title prop:

import { Modal, ModalBody, ModalVariant } from '@patternfly/react-core'; // NEW API
// ...
<Modal title="Custom time range" ...>  // ← OLD API prop, NOT on new Modal
  <ModalHeader title="Custom time range" />  // ← NEW API (correct)

pf-codemods approach

Moves Modal and ModalVariant to @patternfly/react-core/deprecated:

import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';
import { ModalBody } from '@patternfly/react-core';

This makes the old title prop work again, but uses the deprecated API instead of completing the migration to the new API. Also leaves a dead commented-out import.

semver/goose approach

Removes the title prop from <Modal>, keeping the new API import:

-      title="Custom time range"

The <ModalHeader title="Custom time range"> already provides the title via the new API. This is the correct forward-looking fix.

🖱️ Popper onDocumentClick Removal Winner: semver/goose

What changed

While onDocumentClick is still present in the PF6 Popper type definitions (v6.4.1), the semver/goose branch proactively replaces it with a manual useEffect-based click handler. This is a forward-compatibility improvement — the prop is on a deprecation trajectory and the manual approach is more explicit.

semver/goose implementation

React.useEffect(() => {
  const onDocumentClick = (event: MouseEvent) => {
    if (inputRef.current !== event?.target &&
        !menuRef.current?.contains(event?.target as HTMLElement)) {
      closeMenu();
    }
  };
  if (isMenuOpen) {
    document.addEventListener('click', onDocumentClick);
  }
  return () => { document.removeEventListener('click', onDocumentClick); };
}, [isMenuOpen, closeMenu]);
