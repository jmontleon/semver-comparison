📋 Modal Deprecation pf-codemods WINS (partial)

What Changed in PF6

The legacy Modal API is deprecated. It's moved to @patternfly/react-core/deprecated. The new composable Modal (with ModalHeader, ModalBody, ModalFooter) is promoted from @patternfly/react-core/next to @patternfly/react-core.

The full migration requires refactoring from the legacy API to the new composable API. Using the /deprecated import path is an acceptable transitional step.

Aspectpf-codemodssemver/goose
Modal import updated?Partially — moved to /deprecated pathNo change
Composable Modal refactor?NoNo
File affectedTimeRangeDropdown.tsx—

pf-codemods at least acknowledged the deprecation by moving the import. This is a valid transitional step per the PF6 Upgrade Guide. semver/goose left the Modal import unchanged — however, since Modal was not imported from /deprecated on main either, this means the main branch's Modal usage is actually the new promoted Modal API that was promoted from /next. The pf-codemods branch may have regressed the Modal by moving it to /deprecated when it was already using the promoted version.

⚠️ Critical Nuance

On the main branch, Modal is imported from @patternfly/react-core (the promoted new API). The pf-codemods branch moved it to @patternfly/react-core/deprecated (the old API). This may actually be a regression rather than progress, since the code was already using the new composable children (ModalBody, ModalFooter, ModalHeader).

Both branches still import ModalBody, ModalFooter, ModalHeader from @patternfly/react-core — these are the new API components. The pf-codemods branch creates a confusing split where Modal comes from /deprecated but its children come from the main path.

Given this analysis, pf-codemods' Modal change is at best a wash and at worst a regression. Points awarded are minimal.

🎯 Select onSelect Type Signature semver/goose WINS

What Changed in PF6

The Select component's onSelect callback changed its value parameter type. In PF6, it uses SelectOptionProps['value'] (which is string | number | undefined) instead of a simple string or string | number.

Before (main)

// AdvancedSearchForm.tsx
onSelect={(_e, value: string) => { ... }}

// TimeUnitPicker.tsx
onSelect={(_: ReactMouseEvent<Element, MouseEvent>, value: string | number) => { ... }}

After (semver/goose)

// AdvancedSearchForm.tsx
import { SelectOptionProps } from '@patternfly/react-core';
onSelect={(_e, value: SelectOptionProps['value']) => { ... }}

// TimeUnitPicker.tsx
import { SelectOptionProps } from '@patternfly/react-core';
onSelect={(_: ReactMouseEvent<Element, MouseEvent>, value: SelectOptionProps['value']) => { ... }}
