🪟 Modal Component Migration semver/goose wins

What changed in PF6

The old Modal is deprecated and moved to @patternfly/react-core/deprecated. The new PF6 Modal (promoted from /next) uses a composable API with ModalHeader, ModalBody, and ModalFooter children instead of props like title, showClose, and hasNoBodyWrapper.

pf-codemods approach

- import { Modal, ModalVariant } from '@patternfly/react-core';
+ import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';

// No other changes — same props, same usage pattern

Moves Modal to the deprecated import path. This is the minimum codemod output — it delays migration rather than completing it. Since PF5 packages are still installed, the /deprecated subpath doesn't even exist, causing 18 build errors.

semver/goose approach

+ import { Modal, ModalHeader, ModalVariant } from '@patternfly/react-core';

// Before:
- <Modal title={t('Edit site')} variant={ModalVariant.medium}
-   isOpen={isModalOpen} showClose={false}>

// After:
+ <Modal variant={ModalVariant.medium} isOpen={isModalOpen}>
+   <ModalHeader title={t('Edit site')} />

Uses the new PF6 composable Modal API. Removes deprecated props (title, showClose, hasNoBodyWrapper) and adds ModalHeader as a child component. Applied across all 10 Modal usages.

Build impact

However, both branches still have build errors for Modal. semver/goose's errors are for leftover showClose and hasNoBodyWrapper props that weren't fully removed in some files (11 type errors). pf-codemods has 18 import errors because /deprecated doesn't exist in PF5.

📭 EmptyState Restructuring semver/goose wins

What changed in PF6

EmptyStateHeader and EmptyStateIcon are no longer exported. Instead, use titleText, headingLevel, and icon props directly on <EmptyState>.

pf-codemods approach

+     {message || t('Data not found')}
+   </Title>} variant={EmptyStateVariant.lg}>

// Loading/index.tsx — whitespace artifacts:

Partially applies the restructuring. EmptyStateHeader is removed but EmptyStateIcon remains as a child in EmptyData. Wraps a full <Title> JSX element inside titleText (non-idiomatic). Double-space formatting artifacts from automated tool.

semver/goose approach

// EmptyData/index.tsx — clean prop migration:
+ <EmptyState variant={EmptyStateVariant.lg} isFullHeight
+   data-testid="sk-emptydata" icon={icon}>
+   <Title headingLevel="h2" size="lg">

// Loading/index.tsx — clean:
+ <EmptyState icon={Spinner}>

Clean prop-based migration. icon moved to prop on EmptyState. No formatting artifacts. <Title> left as a child element (readable).

Build impact

Both branches have 4 EmptyState-related build errors (EmptyStateIcon/EmptyStateHeader still imported). semver/goose's approach is cleaner but not fully complete either.

📄 PageSection Variant Migration semver/goose wins

What changed in PF6

PageSectionVariants.light is removed. Valid variants are now "default" or "secondary". PageNavigation component was removed entirely.

pf-codemods approach

- <PageSection variant={PageSectionVariants.light}>
+ <PageSection hasBodyWrapper={false}>

Replaces variant={light} with hasBodyWrapper={false}. This is semantically incorrect — hasBodyWrapper is a different prop that controls whether a body wrapper div is rendered, not the visual variant. The 8 resulting type errors confirm hasBodyWrapper doesn't exist on PF5's PageSection.

semver/goose approach

- <PageSection variant={PageSectionVariants.light}>
+ <PageSection variant={PageSectionVariants.default}>

Correctly maps light → default per PF6 guidance. However, still produces build errors because PageSectionVariants enum may have changed shape in PF6. The mapping is correct per official docs.

Both branches correctly remove PageNavigation.
