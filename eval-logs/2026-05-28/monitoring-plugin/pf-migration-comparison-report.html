6.2 EmptyState Migration Critical Gap on B1

What changed in PF6

EmptyState underwent major refactoring in PF6:

  • EmptyStateHeader and EmptyStateIcon are no longer separate components
  • titleText should be a plain string (not a <Title> element), with headingLevel as a separate prop
  • icon prop takes the icon directly (as element in some cases, component in others)
  • EmptyStateFooter and EmptyStateActions wrap buttons

PF6 Correct Pattern

<EmptyState
  titleText="My Title"
  headingLevel="h4"
  icon={MyIcon}
  variant={EmptyStateVariant.sm}
>
  <EmptyStateBody>...</EmptyStateBody>
  <EmptyStateFooter>
    <EmptyStateActions>...</EmptyStateActions>
  </EmptyStateFooter>
</EmptyState>

What changed in PF6

  • The old Modal is deprecated → moved to @patternfly/react-core/deprecated
  • The new composable Modal (from /next) is promoted to @patternfly/react-core
  • With the new Modal: the title prop does NOT exist — title goes in <ModalHeader title="..." />

Branch Comparison

pf-codemods-053126-0326

Strategy: Moved Modal + ModalVariant imports to @patternfly/react-core/deprecated in 5 files.

Bug in SilencesUtils.tsx:

<Modal isOpen={isOpen} position="top"
  title={t('Expire silence')}          ← OLD deprecated prop
  variant={ModalVariant.small}>
  <ModalHeader title={t('Expire Silence')} />  ← NEW prop
  <ModalBody>...

The title prop on <Modal> is duplicated with the <ModalHeader>. Using the deprecated import means the old title prop is active, so the title will render twice — once from Modal's old rendering and once from ModalHeader.

Also a strategic concern: Using /deprecated imports is a backward step. These Modals already used the new composable API (ModalHeader/ModalBody/ModalFooter), so moving them to deprecated is wrong — they should stay on the promoted API.

6.4 Button Icon Prop Migration

What changed in PF6

In PF6, Button icons must use the icon prop instead of being passed as children. The <Icon> wrapper component is also removed — icons are passed directly as elements.

// PF5 (old)
<Button variant="plain"><EllipsisVIcon /></Button>
<Button><Icon size="sm"><PlusCircleIcon /></Icon> Add</Button>

// PF6 (new)
<Button variant="plain" icon={<EllipsisVIcon />} />
<Button icon={<PlusCircleIcon />}>Add</Button>
