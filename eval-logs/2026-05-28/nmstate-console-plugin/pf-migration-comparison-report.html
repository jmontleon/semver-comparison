⚠️ Layout Components (StackItem, SplitItem) CRITICAL: semver/goose introduces runtime crashes

What PF6 requires

StackItem and SplitItem have a component prop that sets the HTML element type to render (defaults to 'div'). It expects a string like 'div', 'span', 'section' — NOT a React element.

PF6 Source Code (authoritative)

// @patternfly/react-core/src/layouts/Stack/StackItem.tsx
export const StackItem = ({
  component = 'div',  // ← DEFAULT IS STRING 'div'
  ...props
}) => {
  const Component = component as any;
  return <Component {...props}>{children}</Component>;  // ← USED AS ELEMENT TYPE
};
CRITICAL BUG in semver/goose: Branch 2 passes JSX elements (React nodes) to the component prop instead of element type strings. This pattern will crash at runtime because React cannot use a rendered element as a component constructor. There are 20+ instances of this pattern across ~10 files.

semver/goose pattern (WRONG — will crash)

// BEFORE (correct — children as content)
<StackItem>
  <Radio id="..." label="..." />
</StackItem>

// AFTER in semver/goose (WRONG — JSX element as component type)
<StackItem component={<Radio id="..." label="..." />} />

// What component prop actually expects:
<StackItem component="span">  {/* ← string element type */}
  <Radio id="..." label="..." />
</StackItem>
