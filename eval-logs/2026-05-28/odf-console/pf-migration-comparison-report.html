🪟 Modal Migration (deprecated → new PF6 API) — CRITICAL

What changed in PF6

The old Modal component (with title, actions, description props) was deprecated. The new Modal from /next was promoted to main export. New API uses composable <ModalHeader>, <ModalBody>, <ModalFooter> children.

Before (PF5 deprecated Modal)

import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';
<Modal title="Title" actions={[...]} description="...">
  {body content}
</Modal>

After (PF6 new Modal)

import { Modal, ModalVariant, ModalHeader, ModalBody, ModalFooter } from '@patternfly/react-core';
<Modal isOpen onClose={...}>
  <ModalHeader title="Title" />
  <ModalBody>{body content}</ModalBody>
  <ModalFooter>{actions}</ModalFooter>
</Modal>

🔘 Button icon Prop Migration — BUG ALERT

What changed in PF6

Icons should be passed to the icon prop instead of as children for variant="plain" buttons. The icon prop expects a React element (e.g. <SearchIcon />), not a text string.

Correct PF6 usage

<Button variant="plain" icon={<SearchIcon />} aria-label="Search" />

What codemods did (WRONG)

// Before (main) — text as children, correct for non-plain buttons
<Button variant="plain" onClick={toggleShowSecret}>
  {t('Switch to Secret')}
</Button>

// After (codemods) — text string in icon prop, INCORRECT
<Button icon={t('Switch to Secret')} variant="plain" onClick={toggleShowSecret} />
🐛 BUG: The codemods branch introduced 5 new instances of icon={t('text string')}. This passes a translated text string to the icon prop, which expects a React icon element. These buttons will either render the text as an icon (incorrect styling) or break visually. The icon prop migration rule only applies to icon-only buttons, not text buttons.
