🪟 Modal Migration (Deprecated → Composable PF6 API)

What changed in PF6

The old props-based Modal was deprecated in PF5 and moved to @patternfly/react-core/deprecated. In PF6, the "next" Modal was promoted to the main export. The new Modal uses a composable API with ModalHeader, ModalBody, and ModalFooter children.

Before (main branch — ConfirmationModal.tsx)

import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';

<Modal
  actions={[<Button ...>Leave</Button>, <Button ...>Stay</Button>]}
  title={t('Do you want to leave this page?')}
  titleIconVariant="warning"
  showClose={false}
  variant={ModalVariant.small}
>
  {t('Changes you made may not be saved.')}
</Modal>

After (fix-engine — ConfirmationModal.tsx)

import { Button, Modal, ModalBody, ModalFooter, ModalHeader, ModalVariant } from '@patternfly/react-core';

<Modal isOpen={true} variant={ModalVariant.small}>
  <ModalHeader title={t('Do you want to leave this page?')} titleIconVariant="warning" />
  <ModalBody>{t('Changes you made may not be saved.')}</ModalBody>
  <ModalFooter>
    <Button ...>Leave</Button>
    <Button ...>Stay</Button>
