🔲 Modal Migration fix-engine pf-codemods REGRESSION

What changed in PF6

In PF6, the old Modal (which used props like title=, actions=, footer=) was deprecated and moved to @patternfly/react-core/deprecated. The new promoted Modal uses a composition pattern with <ModalHeader>, <ModalBody>, <ModalFooter> as children. ModalVariant is exported from both the promoted and deprecated paths.

Critical Finding

⚠️ pf-codemods made an incorrect change: On main, the project already uses the promoted PF6 Modal pattern (with ModalHeader, ModalBody, ModalFooter as children imported from @patternfly/react-core). The codemod tool incorrectly moved Modal and ModalVariant to @patternfly/react-core/deprecated across 40 files. This is a regression — it switches correct PF6 code back to the deprecated API path.

Before (main — correct PF6 usage):

import {
  Modal, ModalBody, ModalFooter, ModalHeader, ModalVariant
} from '@patternfly/react-core';

<Modal isOpen={isOpen} onClose={closeModal} variant={ModalVariant.small}>
  <ModalHeader title={headerText} />
  <ModalBody>{children}</ModalBody>
  <ModalFooter>...</ModalFooter>
</Modal>

After (pf-codemods — incorrect regression):

import { ModalBody, ModalFooter, ModalHeader } from '@patternfly/react-core';
import { Modal, ModalVariant } from '@patternfly/react-core/deprecated'; // ❌ WRONG

// Same JSX usage — but now importing from deprecated path

fix-engine approach (correct):

import {
  Modal, ModalBody, ModalFooter, ModalHeader, ModalVariant
} from '@patternfly/react-core'; // ✅ Promoted path

// Migrated old-style Modal (title=, actions= props) to composition pattern:
<Modal>
  <ModalHeader title={title} />
  <ModalBody>...</ModalBody>
  <ModalFooter>...</ModalFooter>
</Modal>
