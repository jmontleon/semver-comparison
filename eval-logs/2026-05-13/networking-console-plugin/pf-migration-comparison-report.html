Key Risk Analysis

⚠️ fix-engine: PF5/PF6 Package Mixing Risk

fix-engine creates a dangerous package alignment state:

  • @patternfly/react-core: ^6.4.1 (PF6)
  • @patternfly/react-charts: 7.2.2 (PF5-era)
  • @patternfly/quickstarts: 5.1.0 (PF5-era)
  • @patternfly/react-catalog-view-extension: 5.0.0 (PF5-era)
  • @patternfly/react-log-viewer: 5.1.0 (PF5-era)

These PF5-era packages may ship their own CSS that references pf-v5-* class names, include deprecated component APIs, or have peer dependency conflicts with react-core v6. This is explicitly warned against in the PatternFly upgrade guide: all @patternfly/* packages should be at the same major generation.

⚠️ fix-engine: SDK Downgrade Impact

Downgrading @openshift-console/dynamic-plugin-sdk from 4.20.0 to 1.3.0 is a massive regression:

  • SDK 4.20 is designed for PF6 and OpenShift 4.20+
  • SDK 1.3 is from the PF4/early PF5 era
  • This forced removal of types like ClusterUserDefinedNetworkKind, UserDefinedNetworkLocalnet
  • This forced replacement with semantically wrong types (e.g., RouteKind for UDN filters)
  • This breaks integration with current OpenShift console versions

This is not a PF migration change — it's an SDK downgrade that actively harms compatibility.
