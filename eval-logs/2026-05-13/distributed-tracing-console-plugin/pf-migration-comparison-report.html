6.2 PageSection / hasBodyWrapper Migration pf-codemods wins

What Changed in PF6

In PatternFly 6, <PageSection> defaults to wrapping content in a body wrapper element. To preserve PF5 behavior, you must add hasBodyWrapper={false}. This is the primary remaining migration task the codemods identified.

pf-codemods approach (correct)

Applied the standard pf-codemods transformation to all 9 <PageSection> instances across 3 files:

- <PageSection>
+ <PageSection hasBodyWrapper={false}>

Files modified:

  • TraceDetailPage.tsx — 1 instance
  • QueryBrowser.tsx — 2 instances
  • TracesPage.tsx — 6 instances

The existing instance at TraceDetailPage.tsx:69 that already had hasBodyWrapper={false} was correctly left unchanged.

fix-engine approach (divergent)

Instead of adding hasBodyWrapper={false}, fix-engine restructured the page layout entirely. It uses <PageSection variant="secondary"> instead of the bare <PageSection>. While this doesn't trigger a build error, it represents a different layout approach that isn't the standard PF6 codemod output and changes the visual appearance.

- <PageSection>
+ <PageSection variant="secondary">

Notably, fix-engine does not use hasBodyWrapper={false} at all — the prop appears zero times in the fix-engine branch's web/src/ files.

Verdict: pf-codemods applies the official codemod output exactly as recommended. fix-engine takes a different architectural approach that mixes layout changes with the migration.

6.3 CSS / Token Migration pf-codemods wins

What Changed in PF6

PF6 introduced a new design token system. The old --pf-global--* variables are replaced by --pf-t--global--* tokens. CSS class prefixes changed from pf-v5- to pf-v6-, and old unversioned prefixes like pf-c- are PF4-era and completely invalid.

main (base) — already PF6 compliant

Main branch CSS files already use PF6 design tokens:

/* PersesWrapper.css — PF6 tokens ✅ */

border: var(--pf-t--global--border--width--action--default) solid var(--pf-t--global--border--color--default);

/* ScatterPlot.tsx — PF6 tokens ✅ */
border: 'var(--pf-t--global--border--width--box--default) solid var(--pf-t--global--border--color--default)'

pf-codemods — preserves PF6 compliance ✅

No CSS changes. Inherits main's already-correct PF6 tokens.

fix-engine — introduces PF5/PF4 tokens ❌
