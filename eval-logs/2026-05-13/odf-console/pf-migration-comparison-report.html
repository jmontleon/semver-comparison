🔄 Modal Migration (deprecated → composable) fix-engine wins

What changed in PF6

The old Modal (with title, description, onClose as props) was deprecated. The new composable Modal uses <ModalHeader>, <ModalBody>, <ModalFooter> as children. Old Modal is available at @patternfly/react-core/deprecated.

pf-codemods approach — ❌ REGRESSION

- import { Modal, ModalVariant, ModalHeader, ModalBody, ... } from '@patternfly/react-core';
+ import { ModalHeader, ModalBody, ... } from '@patternfly/react-core';
+ import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';

This is a regression. The CreateDRPolicyModal.tsx file on main already uses the new composable Modal (imported from @patternfly/react-core) with <ModalHeader> and <ModalBody> as children. The codemod moved Modal backwards to the deprecated import, while leaving the new ModalHeader/ModalBody children in place. This creates a broken mix: the deprecated Modal expects title/description as props, not composable children. The result is a Modal with no header rendered via the deprecated path, and misplaced ModalHeader/ModalBody children inside the deprecated Modal body.

🔘 Button icon Prop Migration fix-engine wins

What changed in PF6

For variant="plain" buttons with icon content, the icon should be passed to the icon prop instead of as children. However, text-only buttons should keep text as children. The icon prop expects a React element (icon component), not a text string.

pf-codemods — ❌ INCORRECT

- <Button variant="plain" onClick={switchToSecret}>
-   {t('Switch to Secret')}
- </Button>
+ <Button icon={t('Switch to Secret')} variant="plain" onClick={switchToSecret} />

The codemod incorrectly moved text content to the icon prop. The icon prop expects a React icon component, not a string. This results in:

  • Button text ("Switch to Secret", "Switch to Credentials", "Cancel", etc.) rendered in the icon slot
  • Likely visual breakage — text may render with wrong styling or not at all
  • 6 instances across 3 files

fix-engine — ✅ CORRECT

<Button variant="plain" onClick={switchToSecret}>
  {t('Switch to Secret')}
</Button>

Preserved text as children — correct behavior. These buttons have text content, not icons, so no migration was needed.

This is the most significant bug in pf-codemods. The automated codemod incorrectly identified text children as icons. This would cause visible UI regression in the S3/MCG endpoint configuration, GCP endpoint configuration, and IAM user creation flows.
