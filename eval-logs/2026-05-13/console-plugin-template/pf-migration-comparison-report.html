📄 PageSection Variant Migration

What changed in PF6

In PF6, variant="light" and variant="dark" were removed. The PF6 PageSection only accepts variant="default" or variant="secondary". The default variant (no prop) renders a light/white background — the same visual as PF5's variant="light". The "secondary" variant renders a grey background.

PF5 (base) code:

<PageSection variant="light">...</PageSection>

Correct PF6 migration:

<PageSection>...</PageSection>  <!-- remove variant entirely -->

Official PF6 upgrade PR (a14fab8) by Robb Hamilton:

-  <PageSection variant="light">
+  <PageSection>

🎨 CSS Variable / Token Migration

What changed in PF6

PF6 introduced a completely new token system. Old --pf-global--* and --pf-v5-* CSS variables were replaced with --pf-t--global--* semantic tokens. The old variables are not defined in PF6 CSS at all — they will simply not resolve, causing styles to be missing.

PF5 CSS:

color: var(--pf-global--palette--blue-400);

Correct PF6 CSS:

color: var(--pf-t--global--color--brand--default);

🧩 hasBodyWrapper Prop (New in PF6)

What changed in PF6

PF6 added hasBodyWrapper to PageSection, defaulting to true. This wraps children in a <PageBody> component. In PF5, no such wrapper existed. The official codemods add hasBodyWrapper={false} to preserve PF5 behavior when isWidthLimited was not previously set.
