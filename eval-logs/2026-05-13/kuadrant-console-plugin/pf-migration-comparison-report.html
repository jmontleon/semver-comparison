🔲 Modal Component Migration fix-engine

What changed in PF6

PF6 promotes the composable Modal (previously in @patternfly/react-core/next) to the main import. The old Modal API (title prop, variant as enum) is moved to @patternfly/react-core/deprecated. The new API uses <Modal> as container with <ModalHeader>, <ModalBody>, <ModalFooter> as children. The title prop moves to <ModalHeader title="...">. The variant prop becomes a string ("medium") instead of ModalVariant.medium.

main branch (before)

Mixed: DropdownWithKebab.tsx already uses the composable pattern (with /* data-codemods */ comments). But KuadrantAuthPolicyCreatePage, AddLimitModal, and LimitSelect still use the old Modal API (title prop directly on <Modal>).

pf-codemods — approach: deprecated import path only

// Moved Modal/ModalVariant to deprecated path, but keeps OLD API:
import { Modal, ModalVariant } from '@patternfly/react-core/deprecated';
// Still uses: <Modal title="..." variant={ModalVariant.medium}>
// ModalBody/ModalFooter still imported from main path

This is the minimum codemod output — it avoids a compile error by moving the import, but does not actually migrate to the new API. The Modal will still use the deprecated API at runtime.

fix-engine — approach: full composable migration

// All Modal components imported from main path (new API):
import { Modal, ModalHeader, ModalBody, ModalFooter } from '@patternfly/react-core';
// Uses: <Modal isOpen onClose variant="medium">
//         <ModalHeader title="..." />
//         <ModalBody>...</ModalBody>
//       </Modal>

This is the correct PF6 migration. All 4 Modal usages (AuthPolicy, RateLimitPolicy, AddLimitModal, LimitSelect, DropdownWithKebab) use the new composable API with ModalHeader as a child.

🎨 CSS / Token / Class Migration fix-engine

What changed in PF6

PF6 introduces a new design token system. CSS variables change from --pf-v5-global--* to --pf-t--global--* (design tokens) or --pf-v6-global--* (palette vars). CSS class prefixes change from pf-v5- to pf-v6-. Utility classes should change from pf-u- to pf-v6-u-.

main branch CSS variables

Uses --pf-v6-global--palette--* and --pf-v6-global--Color--200. Some of these are valid PF6 palette variables, but --pf-v6-global--Color--200 does not exist in PF6's CSS (confirmed: not defined in @patternfly/react-styles).

pf-codemods — No CSS changes

Zero CSS file modifications. All --pf-v6-global-- variables remain as-is from main, including the potentially invalid --pf-v6-global--Color--200.

fix-engine — Partial token migration
