🎯 Popper Component Removal

fix-engine Fully removed

Removed all Popper component usage from 4 dropdown files (overview-display, query-options, table-display, topology-display). Replaced Popper + useCallback pattern with simple conditional rendering:

Before (PF5 — Popper)
import { Button, Popper } from '@patternfly/react-core';
const triggerRef = React.useRef(null);
const popperRef = React.useRef(null);
const trigger = React.useCallback(() => (
  <Button ref={triggerRef} ...>{t('Options')}</Button>
), [...]);
const popper = React.useCallback(() => (
  <div ref={popperRef} className="pf-v5-c-menu">...</div>
), [...]);
<Popper trigger={trigger()} popper={popper()}
  isVisible={isOpen} appendTo={...} />
After (fix-engine)
import { Button } from '@patternfly/react-core';
// No refs, no useCallback wrappers needed
<Button variant="link" onClick={() => setOpen(!isOpen)}>
  {t('Options')}
</Button>
{isOpen && (
  <div className="pf-v6-c-menu" role="dialog">
    <OptionsPanel {...props} />
  </div>
)}

pf-codemods-051326-2137 Still uses Popper

Popper remains in 5 files. This component is deprecated in PF6 and will be removed in a future version.
